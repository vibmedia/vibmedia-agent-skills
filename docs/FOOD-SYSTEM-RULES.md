# Food System Rules & Architecture

> This document acts as the persistent memory and guardrails for the `D:\food` image processing workflow to be retained across different computers via Google Drive workspace sync.

---

## 📂 Core Folder Architecture
- **`raw/`**: Where original / unprocessed images of food are dropped. (e.g., `SKU-name.jpg`)
- **`final/`**: Where post-processed master photos reside. The filenames MUST match the `System Item` in the CSV exact matching.
- **`[Brand Name]/`**: Auto-generated by Python Script.
  - **`brand-raw/[Brand Category]/[Brand Item]/`**: Receives copies of the images from `final/`.
  - **`ready/[Brand Category]/[Brand Item]/`**: Where brand-specific ready-to-use images with logos should be saved manually after designers edit them.

## 📝 CSV Format & Execution
To route the images correctly, supply a CSV mapping to the `process_brand.py` script.
**Required exact column names:**
1. `System Category`
2. `Brand Category`
3. `System Item` (Matches final image filename exactly, without extension)
4. `Brand Item` (Dictates final folder and assigned image name)

**Running the automation:**
Open a terminal (PowerShell or Bash) and run:
```bash
cd D:\food
python .agent\scripts\process_brand.py "Brand Name" mapping.csv
```

## 🛡️ Guardrails
- **Manual Execution Strategy:** We explicitly use a manual command-line script instead of background watcher-daemons. This maintains absolute predictability, is lightweight, and easily portable between computers.
- **File Matching Logic:** Script actively scans `final/` directory for `.jpg`, `.png`, `.jpeg`, and `.webp` extensions matching the `System Item` name. If a match isn't found, it prints a warning and skips.
- **No Over-engineering:** Only use standard Python libraries (`os`, `pathlib`, `csv`, `shutil`) to ensure the scripts can run on any environment without `pip install` prerequisites.

## 📥 Intake Processing Strategy
- **`intake/`**: A staging directory where unorganized external image drops are placed. 
- **`process_intake.py`**: Fuzzy-matches filenames in `intake/` against the `mapping.csv`. Matching files are moved and renamed to `[System Item].ext` and placed directly into `raw/`. 
  - **Auto-Generation:** If a file cannot be confidently matched to the script but has a clean, descriptive name (e.g., letters and spaces only, like 'Hot and Sour Soup.jpeg'), it is automatically moved to `raw/`, and a new entry `[Uncategorized, "", "Title Cased Name", ""]` is appended to the `.csv`.
  - Confusing or strictly random names (e.g., `IMG_001.jpg`) are intentionally left in `intake/` for manual review.
  - **Usage:** Run `python .agent\scripts\process_intake.py mapping.csv`
