# Profile: Hybrid (Dev + Marketing)

> Load this profile when `context.md` has `profile: hybrid`.
> Both dev and marketing agents/skills are active, with phase-based priority.

---

## How Hybrid Works

Hybrid projects typically have two phases:

```
Phase 1: BUILD → Dev agents lead, marketing agents advise
Phase 2: GROW  → Marketing agents lead, dev agents support
```

The current phase is set in the brand's `todo.md`:
- If most planned tasks are dev (build, code, deploy) → Phase 1 behavior
- If most planned tasks are marketing (campaign, content, SEO) → Phase 2 behavior

## Hybrid-Specific Rules

1. **Don't mix phases in one session** — focus on either building or marketing
2. **Dev decisions should consider marketing** — URL structure, meta tags, analytics hooks
3. **Marketing should reference the product** — know what's built and what's planned
4. **Use `orchestrator`** when a task spans both domains
